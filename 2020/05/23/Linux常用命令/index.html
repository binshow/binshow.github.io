22.CPU过高，如何定位

结合Linux和JDX命令块一起分析

  1. top命令找到cpu占比最高的
  2. ps -ef 或者jps进一步定位 ，得知是怎么样的一共后台程序
  3. 定位到具体线程或代码 ps -mp 进程 -o THREAD,tid,time
  4. 将需要的线程ID切换成16进制的格式（英文小写格式）
  5. jstack 进程ID | grep tid（16进制线程ID小写的英文） -A60 打印出前60行

ps -mp 进程 -o THREAD,tid,time 中

-m显示所有的线程

-p pid进程使用CPU的时间

-o 该参数后是用户自定义格式
